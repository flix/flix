# WIT Op Catalog

Status: source-of-truth boundary catalog  
Last updated: 2026-03-10

This document is a **living catalog** that maps **portable Flix primops** (and their current JVM
semantics) to the **typed per-op WIT surface** used for wasm embedding (browser + WASI 0.3/P3).

Goal: make it hard for the runtime/host bridge to drift from the portable stdlib + JVM behavior.

Non-goal: define the internal lowering or runtime implementation (this is about the boundary).

---

## 0. Terminology and Invariants

- A **suspension** represents a blocked Flix computation (effect op / syscall / async I/O).
- The host observes the requested operation via `suspension-request: op-request`.
- The host resumes the computation by calling the corresponding **typed** `resume-*` export.
- The host must resume each suspension **exactly once** (the `suspension` resource is consumed).

Notes:

- Typed per-op resumers exist so the host does **not** need to fabricate Flix `value` handles for
  syscall results.
- Error payloads use `io-error { kind-code, msg }` where `kind-code` matches
  `IoError.errorKindFromCode` (portable stdlib convention).
- In JS bindings generated by `jco`, `list<u8>` is represented as `Uint8Array` (including in
  resumer parameters like `resume-tcp-socket-read-ok`).
- In JS bindings generated by `jco`, `list<string>` is represented as `string[]`.

## 1. Timer

### 1.1 `%%SLEEP_MILLIS%%` / `IoOp.SleepMillis`

Flix surface (portable stdlib):

- `SleepMillis` is used for `Thread.sleep`-like behavior in portable code.

WIT:

- `op-request.timer-sleep(timer-sleep-req { ms })`
- `resume-timer-sleep(ctx, s)`

Semantics:

- `resume-timer-sleep` unblocks the suspended computation (no payload; avoids “fabricate Unit”).

## 2. HTTP

### 2.1 `%%HTTP_REQUEST%%` / `Http.request`

Flix surface (portable stdlib):

- `%%HTTP_REQUEST%%((method, url, reqHeaders, hasBody, bodyStr))`
- Result tuple: `(ok, status, respPairs, respBody, kindCode, msg)`

WIT:

- Request:
  - `op-request.http-request(http-request-req { method, url, headers, body })`
- Resume:
  - `resume-http-ok(ctx, s, http-response { status, headers, body })`
  - `resume-http-err(ctx, s, io-error { kind-code, msg })`

Semantics to preserve:

- Redirect behavior and timeouts are defined by portable JVM tests (see `PortableHttpSuite.scala`).
- Invalid UTF-8 response bodies are decoded **lossily** (U+FFFD replacement); this is implemented
  in the runtime string layer, not the host bridge.

## 3. TCP Sockets (Portable)

TCP in portable Flix is represented using **opaque integer handles** (`Int64`) that the runtime
passes to host-backed operations.

WIT treats these handles as `u64` **opaque ids**:

- ids are generated by the host (connect/accept),
- ids are only meaningful within the owning `ctx`,
- ids should not be reused until `ctx` is dropped (simplifies validity tracking).

### 3.1 `%%TCP_SOCKET_CONNECT%%` / `TcpConnect.connect`

Flix surface:

- `%%TCP_SOCKET_CONNECT%%((ipBytes, port))`
- Result tuple: `(ok, socketId, kindCode, msg)`

WIT:

- Request:
  - `op-request.tcp-socket-connect(tcp-socket-connect-req { ip, port })`
- Resume:
  - `resume-tcp-socket-connect-ok(ctx, s, socket-id: u64)`
  - `resume-tcp-socket-connect-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve (high-level):

- Invalid address bytes ⇒ `InvalidInput`.
- I/O failure ⇒ `Other`.

### 3.2 `%%TCP_SOCKET_READ%%` / `TcpSocket.read`

Flix surface:

- `%%TCP_SOCKET_READ%%((socketId, buffer))`
- Result tuple: `(ok, n, msg)`

WIT:

- Request:
  - `op-request.tcp-socket-read(tcp-socket-read-req { socket-id, max-bytes })`
- Resume:
  - `resume-tcp-socket-read-ok(ctx, s, bytes: list<u8>)`
  - `resume-tcp-socket-read-err(ctx, s, io-error { kind-code, msg })`

Semantics to preserve:

- `bytes.len` becomes the returned `n`.
- EOF is represented as `bytes = []` and yields `n = 0` (JVM maps `-1` → `0`).
- On error, portable stdlib currently maps to `IoError(ErrorKind.Other, msg)`; `kind-code` is
  accepted for future-proofing but may be ignored by the runtime for this primop.

### 3.3 `%%TCP_SOCKET_WRITE%%` / `TcpSocket.write`

Flix surface:

- `%%TCP_SOCKET_WRITE%%((socketId, buffer))`
- Result tuple: `(ok, n, msg)`

WIT:

- Request:
  - `op-request.tcp-socket-write(tcp-socket-write-req { socket-id, bytes })`
- Resume:
  - `resume-tcp-socket-write-ok(ctx, s)`
  - `resume-tcp-socket-write-err(ctx, s, io-error { kind-code, msg })`

Semantics to preserve:

- “Write all or error” (JVM writes the whole buffer or throws).
- On success, the runtime sets `n = bytes.len` (host does not provide `n`).

### 3.4 `%%TCP_SOCKET_CLOSE%%` / `TcpSocket.close`

Flix surface:

- `%%TCP_SOCKET_CLOSE%%(socketId)`
- Result tuple: `(ok, msg)`

WIT:

- Request:
  - `op-request.tcp-socket-close(tcp-socket-close-req { socket-id })`
- Resume:
  - `resume-tcp-socket-close-ok(ctx, s)`
  - `resume-tcp-socket-close-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve:

- Closing an “unknown” handle is treated as **success** (`(true, "")`).

## 5. Filesystem (Portable)

Portable file I/O in Flix is currently exposed via `%%FILE_*%%` primops used by
`main/src/library/portable/FileRead*.flix` and `main/src/library/portable/FileWrite*.flix`.

All primops return an `ok: Bool` plus either:

- a success value (`Bool` / `Int64` / `String` / `Array[...]`), or
- a portable error payload (`kindCode: Int32`, `msg: String`) where `kindCode` matches
  `IoError.errorKindFromCode`.

WIT uses typed-per-op requests + typed resumers so the host does not fabricate Flix `value` handles.

### 5.1 Predicates

Applies to:

- `%%FILE_EXISTS%%`
- `%%FILE_IS_DIRECTORY%%`
- `%%FILE_IS_REGULAR_FILE%%`
- `%%FILE_IS_READABLE%%`
- `%%FILE_IS_SYMBOLIC_LINK%%`
- `%%FILE_IS_WRITABLE%%`
- `%%FILE_IS_EXECUTABLE%%`

Flix surface (portable stdlib):

- `(ok, v, kindCode, msg) = %%FILE_...%%(path: String)`

WIT:

- Requests:
  - `op-request.file-exists(file-exists-req { path })`
  - `op-request.file-is-directory(file-is-directory-req { path })`
  - `op-request.file-is-regular-file(file-is-regular-file-req { path })`
  - `op-request.file-is-readable(file-is-readable-req { path })`
  - `op-request.file-is-symbolic-link(file-is-symbolic-link-req { path })`
  - `op-request.file-is-writable(file-is-writable-req { path })`
  - `op-request.file-is-executable(file-is-executable-req { path })`
- Resume:
  - `resume-file-*-ok(ctx, s, v: bool)`
  - `resume-file-*-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve (current behavior):

- `ok=false` only for `InvalidPath` (`kindCode = 3`).
- For non-existing paths, predicates return `ok=true` and `v=false` (not an error).

### 5.2 Metadata (times + size)

Applies to:

- `%%FILE_ACCESS_TIME%%`
- `%%FILE_CREATION_TIME%%`
- `%%FILE_MODIFICATION_TIME%%`
- `%%FILE_SIZE%%`

Flix surface:

- `(ok, v: Int64, kindCode, msg) = %%FILE_...%%(path: String)`
- Time values are **milliseconds since epoch** on JVM.

WIT:

- Requests:
  - `op-request.file-access-time(file-access-time-req { path })`
  - `op-request.file-creation-time(file-creation-time-req { path })`
  - `op-request.file-modification-time(file-modification-time-req { path })`
  - `op-request.file-size(file-size-req { path })`
- Resume:
  - `resume-file-*-ok(ctx, s, ms/bytes: s64)`
  - `resume-file-*-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve (current behavior):

- `InvalidPath` ⇒ `ok=false`, `kindCode=3`.
- Other I/O failures ⇒ `ok=false`, `kindCode=14` (`Other`).

### 5.3 Read

Applies to:

- `%%FILE_READ%%` (read entire file as `String`)
- `%%FILE_READ_LINES%%((rc, path))` (read file as `Array[String]`)
- `%%FILE_READ_BYTES%%((rc, path))` (read file as `Array[Int8]`)

WIT:

- Requests:
  - `op-request.file-read(file-read-req { path })`
  - `op-request.file-read-lines(file-read-lines-req { path })`
  - `op-request.file-read-bytes(file-read-bytes-req { path })`
- Resume:
  - `resume-file-read-ok(ctx, s, data: string)`
  - `resume-file-read-lines-ok(ctx, s, lines: list<string>)`
  - `resume-file-read-bytes-ok(ctx, s, bytes: list<u8>)`
  - corresponding `*-err` resumers with `io-error`

Semantics to preserve:

- JVM uses **UTF-8 (lossy)** for `%%FILE_READ%%` (invalid sequences replaced).
- `%%FILE_READ_LINES%%` returns an array of lines without trailing newlines.
- Errors follow the same `InvalidPath=3` / `Other=14` conventions as above.

### 5.4 List Directory

Applies to:

- `%%FILE_LIST%%((rc, dir))` (list directory entries as `Array[String]`)

WIT:

- Request:
  - `op-request.file-list(file-list-req { path })`
- Resume:
  - `resume-file-list-ok(ctx, s, names: list<string>)`
  - `resume-file-list-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve (current behavior):

- If `path` is not a directory (including “does not exist”), return `NotDirectory` (`kindCode=8`).
- Other failures return `Other` (`kindCode=14`).

### 5.5 Write / Append / Truncate / MkDir

Applies to:

- `%%FILE_WRITE%%((data, path))`
- `%%FILE_WRITE_BYTES%%((bytes, path))`
- `%%FILE_APPEND%%((data, path))`
- `%%FILE_APPEND_BYTES%%((bytes, path))`
- `%%FILE_TRUNCATE%%(path)`
- `%%FILE_MKDIR%%(path)`
- `%%FILE_MKDIRS%%(path)`

WIT:

- Requests:
  - `op-request.file-write(file-write-req { path, data })`
  - `op-request.file-write-bytes(file-write-bytes-req { path, bytes })`
  - `op-request.file-append(file-append-req { path, data })`
  - `op-request.file-append-bytes(file-append-bytes-req { path, bytes })`
  - `op-request.file-truncate(file-truncate-req { path })`
  - `op-request.file-mkdir(file-mkdir-req { path })`
  - `op-request.file-mkdirs(file-mkdirs-req { path })`
- Resume:
  - `resume-file-*-ok(ctx, s)` (unit payload)
  - `resume-file-*-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve (high-level):

- Writes encode strings as UTF-8.
- Append uses “append + create”.
- `mkdir` returns `AlreadyExists` (`kindCode=0`) on existing path.
- `mkdirs` succeeds if the directory already exists; it returns `AlreadyExists` only if the path exists but is not a directory (e.g. a file).
- `Unsupported` maps to `kindCode=12` when the host cannot provide filesystem access.

### 5.6 MkTempDir

Applies to:

- `%%FILE_MK_TEMP_DIR%%(prefix)`

WIT:

- Request:
  - `op-request.file-mk-temp-dir(file-mk-temp-dir-req { prefix })`
- Resume:
  - `resume-file-mk-temp-dir-ok(ctx, s, path: string)`
  - `resume-file-mk-temp-dir-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve (current behavior):

- Illegal prefix values map to `InvalidPath` (`kindCode=3`).

## 6. Process (Host Capability)

Process execution is **not** portable across all wasm hosts (it is not available in browsers and is
not guaranteed in WASI environments). The host must advertise support via `has-capability(Process)`.

If the capability is absent, the runtime must behave like the JVM handlers would for “unsupported”:

- return `ok=false` with `kindCode=12` (`Unsupported`) for the `ProcessWithResult` primops, and
- return `ok=false` with an explanatory message for the legacy `(ok, n, msg)` I/O primops.

WIT provides typed requests + typed resumers for all `%%PROCESS_*%%` primops so hosts do not need to
fabricate Flix `value` handles.

### 6.1 `%%PROCESS_EXEC%%` / `IoOp.ProcessExec`

Flix surface:

- `%%PROCESS_EXEC%%((argvArr, hasCwd, cwdStr, envPairsArr))`
- Result tuple: `(ok, processId, kindCode, msg)`

WIT:

- Request:
  - `op-request.process-exec(process-exec-req { argv, cwd, env })`
- Resume:
  - `resume-process-exec-ok(ctx, s, process-id: u64)`
  - `resume-process-exec-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve:

- `IllegalArgumentException` (invalid argv/cwd/env) ⇒ `InvalidInput` (`kindCode=4`).
- `IOException` ⇒ `Other` (`kindCode=14`).

### 6.2 `%%PROCESS_{EXIT_VALUE,IS_ALIVE,PID,STOP,WAIT_FOR,WAIT_FOR_TIMEOUT}%%`

Flix surface:

- `%%PROCESS_EXIT_VALUE%%(processId)` → `(ok, code: Int32, kindCode, msg)`
- `%%PROCESS_IS_ALIVE%%(processId)` → `(ok, alive: Bool, kindCode, msg)`
- `%%PROCESS_PID%%(processId)` → `(ok, pid: Int64, kindCode, msg)`
- `%%PROCESS_STOP%%(processId)` → `(ok, (), kindCode, msg)`
- `%%PROCESS_WAIT_FOR%%(processId)` → `(ok, code: Int32, kindCode, msg)`
- `%%PROCESS_WAIT_FOR_TIMEOUT%%((processId, timeoutMs))` → `(ok, finished: Bool, kindCode, msg)`

WIT:

- Requests:
  - `op-request.process-exit-value(process-exit-value-req { process-id })`
  - `op-request.process-is-alive(process-is-alive-req { process-id })`
  - `op-request.process-pid(process-pid-req { process-id })`
  - `op-request.process-stop(process-stop-req { process-id })`
  - `op-request.process-wait-for(process-wait-for-req { process-id })`
  - `op-request.process-wait-for-timeout(process-wait-for-timeout-req { process-id, timeout-ms })`
- Resume:
  - `resume-process-*-ok(ctx, s, ...)` with the corresponding typed payload
  - `resume-process-*-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve (high-level):

- Invalid/unknown handle ⇒ `ok=false`, `kindCode=14`, message `"invalid process handle."`.
- `wait-for` / `wait-for-timeout` interruption ⇒ `Interrupted` (`kindCode=2`).
- `pid` unsupported ⇒ `Unsupported` (`kindCode=12`).

### 6.3 `%%PROCESS_{STDIN_WRITE,STDOUT_READ,STDERR_READ}%%`

Flix surface:

- `%%PROCESS_STDIN_WRITE%%((processId, buf))` → `(ok, n: Int32, kindCode: Int32, msg)`
- `%%PROCESS_STDOUT_READ%%((processId, buf))` → `(ok, n: Int32, kindCode: Int32, msg)`
- `%%PROCESS_STDERR_READ%%((processId, buf))` → `(ok, n: Int32, kindCode: Int32, msg)`

WIT:

- Requests:
  - `op-request.process-stdin-write(process-stdin-write-req { process-id, bytes })`
  - `op-request.process-stdout-read(process-stdout-read-req { process-id, max-bytes })`
  - `op-request.process-stderr-read(process-stderr-read-req { process-id, max-bytes })`
- Resume:
  - `resume-process-stdin-write-ok(ctx, s)` / `resume-process-stdin-write-err(ctx, s, io-error { kind-code, msg })`
  - `resume-process-stdout-read-ok(ctx, s, bytes)` / `resume-process-stdout-read-err(...)`
  - `resume-process-stderr-read-ok(ctx, s, bytes)` / `resume-process-stderr-read-err(...)`

Semantics to preserve:

- Invalid/unknown handle ⇒ `ok=false`, `n=0`, `kindCode=14`, message `"invalid process handle."`.
- Cancellation / interruption ⇒ `ok=false`, `n=0`, `kindCode=2`.
- Other I/O failures continue to map through `io-error.kind-code`.

- “Write all or error”: on `stdin-write` success, `n = bytes.len`.
- EOF is represented as `bytes = []` and yields `n = 0` (JVM maps `-1` → `0`).

### 6.4 `%%PROCESS_RELEASE%%` / `IoOp.ProcessRelease`

Flix surface:

- `%%PROCESS_RELEASE%%(processId)` → `(ok, msg)`

WIT:

- Request:
  - `op-request.process-release(process-release-req { process-id })`
- Resume:
  - `resume-process-release-ok(ctx, s)`
  - `resume-process-release-err(ctx, s, io-error { kind-code, msg })`

JVM semantics to preserve:

- Removing an unknown handle is treated as success (`ok=true`).
